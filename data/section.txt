                            <div class="read-more-target coursesummary summary-collapsed"><p data-asw-orgfontsize="16" style="font-size: 16px;">Course Description</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">This course introduces students to the fascinating and highly technical field of software reverse engineering (SRE), equipping them with the knowledge and skills needed to dissect, understand, and analyse binary programs. Students will explore both static and dynamic analysis techniques, mastering industry-standard approaches such as disassembly, decompilation, and automated analysis, while also learning to navigate the challenges posed by obfuscation, anti-analysis, and malware evasion strategies.</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">Through practical exposure to professional tools like IDA, students will gain hands-on experience in analysing malware samples, firmware, and vulnerabilities. The course not only builds expertise in uncovering hidden logic and behaviours within binaries but also empowers students to develop custom tools for tackling emerging security threats.</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">Course Contents</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">The course begins with an overview of software reverse engineering, including static analysis, dynamic analysis, and symbolic execution approaches, and their applications in security and malware research. Students will study instruction set architectures, common file formats, and disassembly algorithms such as linear sweep and recursive descent. Core concepts in binary program analysis—including control flow, data flow, function recognition, and calling conventions—will be explored alongside Windows internals and decompilation challenges.</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">Further modules cover advanced techniques such as anti-disassembly, anti-debugging, and anti-virtual machine strategies, as well as automated hybrid analysis methods like symbolic execution and protocol reverse engineering. Students will also examine software vulnerabilities (e.g., buffer overflows, string format, and type conversion flaws) and exploit techniques, while investigating malware mechanisms, behaviour, detection methods, and obfuscation practices. The course concludes with firmware analysis, highlighting its unique challenges and applications in modern security contexts.</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">Course Objectives</p>
<p data-asw-orgfontsize="16" style="font-size: 16px;">At the end of the course, students should be able to:</p>
<ol data-asw-orgfontsize="16" style="font-size: 16px;">
<li data-asw-orgfontsize="16" style="font-size: 16px;">Provide fundamental problems, principles, and techniques in software reverse engineering of binaries including static analysis techniques, disassembly algorithms, dynamic analysis techniques, automated static and dynamic analysis techniques, malware analysis techniques, anti-analysis techniques, and malware obfuscation and packing techniques</li>
<li data-asw-orgfontsize="16" style="font-size: 16px;">Demonstrate and practise many of the techniques using IDA.</li>
<li data-asw-orgfontsize="16" style="font-size: 16px;">Analyse new malware samples and firmware samples, and develop new analysis tools.</li>
</ol>
